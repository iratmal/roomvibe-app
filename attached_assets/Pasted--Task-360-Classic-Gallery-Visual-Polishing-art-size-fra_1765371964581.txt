## 2) Frames & Depth (No More ‚ÄúStickers‚Äù)

**Problem now:**

* Artworks are rendered as flat planes on the wall.
* They visually read as stickers or decals, not framed canvas.

**What we want:**

* A simple, clean **frame + canvas depth** so it feels like a real painting:

  * Thin frame around the artwork
  * 3D depth (a few cm) so the painting stands slightly off the wall
  * Subtle shadow on the wall from the frame

**Implementation direction:**

In the component where you render the artwork (likely `ArtworkImage`):

1. **Add a shallow box for the canvas** behind the image plane:

```tsx
// Pseudo-code
const FRAME_DEPTH = 0.04; // 4 cm
const CANVAS_OFFSET_FROM_WALL = 0.02; // 2 cm

<group position={[0, 0, CANVAS_OFFSET_FROM_WALL]}>
  {/* Canvas body */}
  <mesh>
    <boxGeometry args={[widthWorld, heightWorld, FRAME_DEPTH]} />
    <meshStandardMaterial
      color="#f5f5f5" // subtle off-white canvas edge
      roughness={0.9}
      metalness={0.0}
    />
  </mesh>

  {/* Image plane on top of canvas */}
  <mesh position={[0, 0, FRAME_DEPTH / 2 + 0.001]}>
    <planeGeometry args={[widthWorld * 0.96, heightWorld * 0.96]} />
    <meshStandardMaterial map={artworkTexture} />
  </mesh>
</group>
```

2. **Add a thin black/dark frame around the canvas**:

   * Either as an inset/outline geometry or a slightly larger box behind the image.

Example idea:

```tsx
const FRAME_THICKNESS = 0.03; // 3 cm border

<mesh>
  <boxGeometry args={[widthWorld + FRAME_THICKNESS, heightWorld + FRAME_THICKNESS, FRAME_DEPTH]} />
  <meshStandardMaterial
    color="#111111"
    roughness={0.4}
    metalness={0.1}
  />
</mesh>
```

3. **Shadows:**

   * Make sure the frame + canvas group `castShadow` and `receiveShadow` are configured so we get a subtle shadow on the wall.

This doesn‚Äôt have to be perfect first try ‚Äì just enough that from the `Center` viewpoint, the artworks clearly look like framed objects, not stickers.

## 3) Movement & Navigation ‚Äì First Step Toward "Street View"

We *don‚Äôt* want to implement full free-fly movement yet, but we want a more natural navigation feeling, closer to ArtSteps or Google Street View.

**Current behavior:**

* We already have the 4 main camera presets: `Entrance`, `Center`, `Back Left`, `Back Right`.
* Movement is done via the buttons at the bottom.

**What we‚Äôd like to add now (minimal first version):**

* **Floor markers / hotspots** that represent these 4 camera positions.
* When user clicks a floor marker, the camera jumps to the corresponding preset (same as clicking the buttons).

**Implementation direction:**

* Create a small component, e.g. `GalleryNavigationMarker`, rendered on the floor at the 4 positions that correspond to the existing camera presets.
* Visually, something simple like:

  * A small round disk on the floor (e.g. teal circle) with subtle opacity
  * Always facing up, not affected by camera rotation

Pseudo-code idea:

```tsx
function NavMarker({ position, label, onClick }) {
  return (
    <mesh
      position={position}
      rotation={[-Math.PI / 2, 0, 0]}
      onClick={onClick}
    >
      <circleGeometry args={[0.4, 32]} />
      <meshStandardMaterial
        color="#2bb7a9"
        transparent
        opacity={0.65}
      />
    </mesh>
  );
}
```

* For now, hook them directly to the existing camera logic, e.g.:

```tsx

/>

<NavMarker
  position={[xCenter, 0.01, zCenter]}
  label="Center"
  onClick={() => setActiveView("center")}
/>
// etc.
```

* Make sure markers are only visible in the 360¬∞ editor / gallery view, not in other parts of the app.

This gives us the first version of a more immersive, "I walk through the space" feeling without adding complex free-movement controls.
